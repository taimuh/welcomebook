# Feature Specification: フロントエンド デザイン改善

**Feature Branch**: `003-fix-frontend-design`
**Created**: 2026-03-02
**Status**: Draft
**Input**: User description: "frontendのデザインを修正します。"

## 背景・概要

現在のフロントエンドは機能的には動作しているが、以下の問題が存在する：

1. **ルートページ（`/`）がNext.jsのデフォルトテンプレートのまま**（Vercelロゴ、"Deploy now"リンクなど）であり、WelcomeBookとして不適切
2. **ゲスト向けページのビジュアルデザインが粗く**、民泊ゲストが初めて訪問した際に不安感を与える可能性がある
3. **モバイル体験が不十分**（ゲストはスマートフォンからアクセスする可能性が高い）

このフィーチャーでは、ゲストが快適に利用できるよう、全体的なデザインを改善する。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ルートページのWelcomeBook化 (Priority: P1)

ゲストがマニュアルのURLを開いたとき（または入力した際）、Next.jsのデフォルトページではなく、WelcomeBookとしてのブランドが伝わるページが表示される。

**Why this priority**: デフォルトのNext.jsテンプレートが表示される状態は製品として未完成であり、信頼性を損なう。最優先で修正が必要。

**Independent Test**: ブラウザで `/` にアクセスし、Next.jsのロゴやVercelへのリンクが表示されず、WelcomeBookのブランドコンテンツが表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** ゲストがルートURL（`/`）にアクセスする、**When** ページが読み込まれる、**Then** Next.jsのデフォルトコンテンツ（ロゴ、"Deploy now"ボタン、"Read our docs"リンク等）が一切表示されない
2. **Given** ゲストがルートURL（`/`）にアクセスする、**When** ページが読み込まれる、**Then** WelcomeBookのブランド名またはサービス説明が表示される
3. **Given** ゲストがルートURL（`/`）にアクセスする、**When** ページが読み込まれる、**Then** ゲストは自分の宿泊施設のマニュアルへの適切な導線（案内）を理解できる

---

### User Story 2 - 物件ページのデザイン改善 (Priority: P2)

ゲストが物件のマニュアルページ（`/[slug]`）を開いたとき、ウェルカム感があり読みやすいデザインで情報を受け取れる。

**Why this priority**: ゲストが最も多く訪問するページであり、第一印象がホスト評価に直結する。

**Independent Test**: 物件スラッグのURLにアクセスし、物件情報が視覚的に整理されて表示され、カテゴリ一覧が直感的に操作できることを確認できる。

**Acceptance Scenarios**:

1. **Given** ゲストが物件ページにアクセスする、**When** ページが読み込まれる、**Then** 物件名が目立つ見出しとして表示される
2. **Given** 物件にウェルカムメッセージが設定されている、**When** ゲストがページを開く、**Then** ウェルカムメッセージが視覚的に強調されて表示される
3. **Given** 緊急連絡先が設定されている、**When** ゲストがページを確認する、**Then** 緊急連絡先が素早く見つけられる場所に目立つデザインで表示される
4. **Given** カテゴリが複数存在する、**When** ゲストがカテゴリ一覧を見る、**Then** 各カテゴリがアイコンと説明付きで視覚的に区別しやすく表示される

---

### User Story 3 - モバイル表示の最適化 (Priority: P3)

ゲストがスマートフォンからマニュアルにアクセスしたとき、文字が適切なサイズで読めて、タップ操作がしやすいレイアウトで表示される。

**Why this priority**: 民泊ゲストは現地でスマートフォンを使う場面が多く、モバイル体験が重要。

**Independent Test**: スマートフォンサイズの画面幅（375px）でページを開き、テキストが読みやすく、ボタン・リンクがタップしやすいサイズで表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** スマートフォンでページにアクセスする、**When** ページが表示される、**Then** 横スクロールが不要で、全コンテンツが縦方向に収まる
2. **Given** スマートフォンでカテゴリ一覧を表示する、**When** ゲストがタップする、**Then** タップ領域が十分に広く（最小44px）、誤タップが起きにくい
3. **Given** スマートフォンでコンテンツ詳細ページを表示する、**When** ゲストが本文を読む、**Then** テキストが拡大なしに読める適切なサイズで表示される

---

### Edge Cases

- カテゴリが0件の場合、空の状態を適切なメッセージで表示する
- 物件名が長い場合（20文字以上）でも、レイアウトが崩れない
- 画像が設定されていない物件でも、適切なフォールバック表示がある
- ゲストが存在しない物件スラッグにアクセスした場合、WelcomeBookブランドのエラーページが表示される（Next.jsデフォルトではない）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ルートページ（`/`）はNext.jsのデフォルトテンプレートコンテンツを一切表示してはならない
- **FR-002**: ルートページはWelcomeBookのブランドを示すコンテンツを表示しなければならない
- **FR-003**: ルートページはゲストに対して、自分の物件マニュアルへのアクセス方法を理解させる導線を提供しなければならない
- **FR-004**: 物件ページのヘッダーは物件名を最も目立つ要素として表示しなければならない
- **FR-005**: ウェルカムメッセージは視覚的に強調されたスタイルで表示されなければならない
- **FR-006**: 緊急連絡先は赤系の警告色を使用し、すぐに目につく位置に表示されなければならない
- **FR-007**: カテゴリ一覧の各アイテムはタップ可能な領域が最小44px×44pxを確保しなければならない
- **FR-008**: 全ページはスマートフォン（375px幅）からPCまでの画面幅で横スクロールなしに表示されなければならない
- **FR-009**: 404エラーページはWelcomeBookのブランドデザインを使用し、ゲストに分かりやすいメッセージを表示しなければならない

### 対象外（スコープ外）

- バックエンドAPI・データモデルへの変更
- 管理画面（Strapi）のデザイン変更

### デザインスコープ（確定）

新しいビジュアルデザインシステムを全面導入する（設計資料 `welcomebook-design-system.html` / `welcomebook-manual-sample.html` に基づく）：
- カラーパレット：和紙（Washi Paper）+ 深インディゴ + ゴールドアクセント
- タイポグラフィ：Shippori Mincho（見出し）+ Noto Sans JP（本文）
- レイアウト：固定トップナビ + 左サイドバー（デスクトップ）+ モバイルドロワー + ボトムタブバー

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ルートページ（`/`）にアクセスしたとき、Next.jsのデフォルトコンテンツ（Vercelロゴ、"Deploy now"テキスト等）が0件表示される
- **SC-002**: スマートフォン（375px幅）で全ページを表示した際、横スクロールが発生しない
- **SC-003**: カテゴリ一覧の各アイテムのタップ領域が、縦横ともに44px以上を確保している
- **SC-004**: 存在しない物件スラッグにアクセスした際、WelcomeBookのブランドを示すページが表示され、Next.jsデフォルトの404ページは表示されない
- **SC-005**: ゲストがマニュアルページを開いた際、物件名が他のテキストより大きく目立つ形式で表示される

## 前提条件・依存関係

### 前提条件

- 既存のTailwind CSS設定を継続利用する
- 既存のコンポーネント構造（PropertyHeader、CategoryList等）を変更するか、スタイルのみを調整するかは実装時に判断する
- バックエンドAPIへの変更は行わない（フロントエンドのみの変更）

### 依存関係

- 機能001（民泊ゲストマニュアルシステム）の実装が完了していること（完了済み）
