# データモデル: 手動テストケース

**ブランチ**: `002-manual-test-cases` | **日付**: 2026-02-18

## 概要

本機能はドキュメント作成のみであるため、データベースエンティティの追加・変更は不要。以下はテストケースドキュメントの論理構造を定義する。

## テストケースドキュメント構造

### テスト実行ヘッダー

| フィールド | 型 | 説明 |
|-----------|-----|------|
| 実行日 | 日付 | テスト実行日 |
| 実行者 | テキスト | テスト担当者名 |
| テスト環境 | テキスト | 実行環境（ローカル/ステージング） |
| ブラウザ | テキスト | 使用ブラウザとバージョン |
| フロントエンドURL | URL | テスト対象のフロントエンドURL |
| 管理画面URL | URL | テスト対象のStrapi管理画面URL |

### テストケース

| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|-----|
| テストID | ID | 一意識別子 | TC-US1-001 |
| タイトル | テキスト | テスト内容の要約 | 物件トップページの表示確認 |
| 対応ストーリー | 参照 | 仕様書のユーザーストーリー | US1 |
| 優先度 | 列挙値 | P1/P2/P3 | P1 |
| 前提条件 | テキスト | テスト実行前に必要な状態 | テスト用物件が作成済み |
| 操作手順 | テキスト | 実行する操作の詳細 | URLにアクセスする |
| 期待結果 | テキスト | 合格基準 | 物件名が表示される |
| 結果 | チェック | 合格/不合格 | [ ] |
| 備考 | テキスト | 不合格時の詳細等 | （任意） |

### テストケースID体系

```text
TC-{カテゴリ}-{連番}

カテゴリ:
  US1 = ユーザーストーリー1（ゲスト閲覧）
  US2 = ユーザーストーリー2（ゲスト検索）
  US3 = ユーザーストーリー3（コンテンツ管理）
  US4 = ユーザーストーリー4（物件管理）
  US5 = ユーザーストーリー5（カテゴリ管理）
  US6 = ユーザーストーリー6（画像管理）
  EC  = エッジケース
  SC  = 成功基準検証
```

### テストデータ構造

**物件データ**

| 項目 | 物件1（通常） | 物件2（空） |
|------|-------------|------------|
| 物件名 | テスト民泊 渋谷 | テスト民泊 空物件 |
| スラッグ | test-shibuya | test-empty |
| 所在地 | 東京都渋谷区1-1-1 | 東京都新宿区2-2-2 |
| ウェルカムメッセージ | ようこそ！快適な滞在をお楽しみください。 | ようこそ！ |
| 緊急連絡先 | 090-1234-5678 | 090-9876-5432 |

**カテゴリデータ（物件1に紐付け）**

| カテゴリ名 | アイコン | 表示順 | コンテンツ数 |
|-----------|---------|--------|------------|
| 設備の使い方 | 📺 | 1 | 2 |
| ハウスルール | 📋 | 2 | 1 |
| 周辺情報 | 🗺️ | 3 | 0（空カテゴリ） |
| 緊急時の対応 | 🚨 | 4 | 0（空カテゴリ） |
| 追加テストカテゴリ | ✨ | 5 | 0（削除テスト用） |

**コンテンツデータ**

| タイトル | カテゴリ | 公開状態 | 本文（概要） |
|---------|---------|---------|-------------|
| エアコンの使い方 | 設備の使い方 | 公開 | リモコンの「運転」ボタンを押してください... |
| エアコンのリモコン操作 | 設備の使い方 | 公開 | リモコンの各ボタンの説明... |
| ゴミの出し方 | ハウスルール | 公開 | 燃えるゴミは月・水・金... |
| Wi-Fiの接続方法 | 設備の使い方 | 下書き | SSID: TestWiFi... |
| チェックアウト手順 | ハウスルール | 公開 | チェックアウトは10時までに... |

**画像テストデータ**

| ファイル名 | 形式 | サイズ | 用途 |
|-----------|------|--------|------|
| test-valid.jpg | JPEG | 500KB | 正常アップロードテスト |
| test-valid.png | PNG | 1MB | PNG形式テスト |
| test-valid.webp | WebP | 300KB | WebP形式テスト |
| test-oversize.jpg | JPEG | 6MB | サイズ制限テスト |
| test-invalid.bmp | BMP | 500KB | 非対応形式テスト |

## リレーション（既存システム参照）

```text
物件 (Property)
  ├── 1:N → カテゴリ (Category)
  │         └── 1:N → コンテンツ (Content)
  └── 1:1 → オーナー (Owner)
```

テストデータはこのリレーションに基づいて作成する。テスト実行前にStrapi管理画面から手動で登録する。
