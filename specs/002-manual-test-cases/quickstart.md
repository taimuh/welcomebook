# クイックスタート: 手動テスト実行ガイド

**ブランチ**: `002-manual-test-cases` | **日付**: 2026-02-18

## 概要

WelcomeBookシステムの手動テストケースを実行するためのガイドです。

## 事前準備

### 1. テスト環境の起動

```bash
# バックエンド（Strapi）起動
cd backend
npm run develop

# フロントエンド（Next.js）起動（別ターミナル）
cd frontend
npm run dev
```

### 2. テストデータの準備

1. ブラウザで `http://localhost:1337/admin` にアクセス
2. 管理者アカウントでログイン
3. `data-model.md` の「テストデータ構造」に記載された以下のデータを登録:
   - 物件2件（テスト民泊 渋谷、テスト民泊 空物件）
   - カテゴリ5件（物件1に紐付け）
   - コンテンツ5件（公開3件、下書き1件含む）
4. テスト用画像ファイルを準備:
   - `test-valid.jpg`（500KB程度のJPEG）
   - `test-valid.png`（1MB程度のPNG）
   - `test-valid.webp`（300KB程度のWebP）
   - `test-oversize.jpg`（6MB以上のJPEG）
   - `test-invalid.bmp`（任意のBMPファイル）

### 3. テスト環境の確認

- フロントエンド: `http://localhost:3000` にアクセスできること
- 管理画面: `http://localhost:1337/admin` にログインできること
- テスト用物件ページ: `http://localhost:3000/properties/test-shibuya` が表示されること

## テスト実行手順

1. `test-cases.md` を開く
2. 「テスト実行情報」セクションに実行日・実行者・環境を記入
3. 各テストケースを上から順に実行:
   - 前提条件を確認
   - 操作手順を実行
   - 期待結果と実際の結果を比較
   - 合格の場合: チェックボックスにチェック `[x]`
   - 不合格の場合: チェックを入れず、備考欄に詳細を記入
4. 全テスト完了後、「テスト結果サマリー」セクションに合計を記入

## テスト実行順序（推奨）

効率的にテストを実行するための推奨順序:

1. **P1テスト（必須）**: TC-US1、TC-US3、TC-US4（基本機能の確認）
2. **P2テスト（重要）**: TC-US2、TC-US5、TC-US6（補助機能の確認）
3. **エッジケース**: TC-EC（異常系の確認）
4. **成功基準検証**: TC-SC（品質基準の確認）

## 不合格時の対応

1. 不合格のテストケースIDと詳細を備考欄に記録
2. 可能であればスクリーンショットを取得
3. 開発チームに報告（テストケースIDを共有）
4. 修正後、該当テストケースのみ再実行
